# Plan Review Evaluation: Task Set

## Design Principles

- 18 goals spanning the academic research assistant domain
- Each goal is self-contained and resettable (read-oriented, no destructive side effects beyond Note/Collection creation)
- Goals vary in tool breadth (1-4 distinct tools required) and step depth (1-6 logical steps)
- Structural failure modes (retry flailing, inter-step data flow, tool misuse, loss of generality) should recur across goals without being specific to any one
- Reset procedure: clear all Notes/Collections created during the trial, reset world model to baseline

## Measures per Goal

- **Steps**: total code blocks generated by planner
- **Failures**: step-level failures, categorized by type (tool error, data flow, retry flail, other)
- **Success**: binary — did the goal produce a correct output artifact?
- **Elapsed time**: wall clock, planner mode
- **LLM calls**: total calls to the planning model

## Experimental Protocol

### Trial A — Baseline
Run all 18 goals in random order. No cached plans, empty world model. Record all measures.

### Trial B — Accumulating Review
Run a *different* random subset of ~9 goals. After each goal completes, perform the full review cycle (review → commit → world model injection). The remaining ~9 goals then run with the accumulated world model but *no cached plans* (force replanning). This separates transfer learning from replay.

### Trial C — Replay
Run all 18 goals using cached reviewed plans where available (from Trial B reviews). Goals not reviewed in Trial B are replanned with the full accumulated world model. Measures the combined practical benefit.

### Analysis
- A vs B-latter-half: transfer learning effect (same goals, one has accumulated learnings)
- A vs C: full system benefit (review + replay + transfer)
- Failure type distributions across trials (does retry flailing disappear after relevant learnings are injected?)

## Task Set

### Tier 1: Single-Tool, Low Complexity (expect 1-2 logical steps)

**G01 — Known Paper Lookup**
Find the paper "Attention Is All You Need" on Semantic Scholar. Create a Note with its title, authors, year, abstract, and citation count.
*Primary tools: semantic-scholar, create-note*

**G02 — Web Fact Retrieval**
Search the web for the current estimated number of parameters in Llama 3 405B. Create a Note with the finding and source URL.
*Primary tools: search-web, create-note*

**G03 — Single Paper Summary**
Fetch the PDF from a given arXiv URL (provide a stable, short paper). Create a Note summarizing the paper's main contribution in 2-3 sentences.
*Primary tools: fetch-text, extract, create-note*

**G04 — Note Discovery**
Find all existing Notes that mention "transformer" or "attention mechanism". Create a Collection containing the matches.
*Primary tools: discover-notes, create-collection*

### Tier 2: Multi-Tool, Moderate Complexity (expect 2-4 logical steps)

**G05 — Literature Search + Filter**
Search Semantic Scholar for papers on "retrieval augmented generation" published after 2023. Filter to papers with >50 citations. Create a Collection of the filtered results.
*Primary tools: semantic-scholar, filter-structured, create-collection*

**G06 — Comparative Web Search**
Search the web for arguments for and against using chain-of-thought prompting in production systems. Create a Note presenting both sides.
*Primary tools: search-web (x2), synthesize, create-note*

**G07 — Extract and Restructure**
Given an existing long Note (pre-seeded, ~3K words of mixed content), extract only the claims about model scaling laws. Create a new Note with just those claims, organized by sub-topic.
*Primary tools: extract, synthesize, create-note*

**G08 — Multi-Paper Citation Check**
Search Semantic Scholar for 3 recent papers on "sparse mixture of experts". For each, extract whether they cite the original "Outrageously Large Neural Networks" (Shazeer et al. 2017) paper. Create a Note summarizing the citation relationships.
*Primary tools: semantic-scholar, map+extract, synthesize, create-note*

**G09 — URL Fetch + Key Claims**
Fetch a specific blog post URL (provide a stable technical blog post). Extract the 3 most important claims. Create a Note with the claims and a one-sentence assessment of evidence quality for each.
*Primary tools: fetch-text, extract, create-note*

**G10 — Cross-Source Dedup**
Search both web and Semantic Scholar for "constitutional AI". Identify papers/sources that appear in both result sets. Create a Note listing the overlapping sources.
*Primary tools: search-web, semantic-scholar, synthesize, create-note*

### Tier 3: Multi-Step Synthesis, Higher Complexity (expect 3-6 logical steps)

**G11 — Research Question Decomposition**
Given the question "How do current LLM agents handle long-horizon planning?", decompose it into 3-4 sub-questions, search Semantic Scholar for each, and create a Collection of relevant papers organized by sub-question.
*Primary tools: synthesize, semantic-scholar (x3-4), create-collection, create-note*

**G12 — Evidence Synthesis from Internal Knowledge**
Given 4-5 pre-seeded Notes on different aspects of RLHF (reward modeling, PPO training, DPO alternatives, human preference collection, alignment tax), synthesize a 500-word essay connecting them. Create a Note with the essay.
*Primary tools: discover-notes, extract (x4-5), synthesize, create-note*

**G13 — Claim Verification Pipeline**
Given the claim "Vision transformers outperform CNNs on all image classification benchmarks as of 2024", search Semantic Scholar and web for supporting and contradicting evidence. Create a Note with a verdict and evidence summary.
*Primary tools: semantic-scholar, search-web, extract, synthesize, create-note*

**G14 — Reading List Construction**
Given the topic "mechanistic interpretability", find 5 foundational papers and 5 recent papers (2024+) on Semantic Scholar. For each, extract a one-sentence summary. Create a Note formatted as an annotated reading list, ordered from foundational to recent.
*Primary tools: semantic-scholar (x2), map+extract, synthesize, create-note*

**G15 — Method Comparison Table**
Search for papers describing RLHF, DPO, and KTO alignment methods. Extract the key characteristics of each (training signal, computational cost, known failure modes). Create a Note with a structured comparison.
*Primary tools: semantic-scholar (x3), map+extract, synthesize, create-note*

**G16 — Gap Analysis**
Search Semantic Scholar for papers on "tool use in language models" from 2023-2024. Extract what capabilities are covered. Compare against a pre-seeded Note listing desired agent tool-use capabilities. Create a Note identifying gaps not yet addressed in the literature.
*Primary tools: semantic-scholar, extract, discover-notes, synthesize, create-note*

**G17 — Trend Analysis**
Search Semantic Scholar for papers on "prompt engineering" from 2021, 2022, 2023, and 2024 separately. Count results and extract dominant sub-topics for each year. Create a Note describing how the field's focus has shifted.
*Primary tools: semantic-scholar (x4), map+extract, synthesize, create-note*

**G18 — Multi-Hop Reasoning Chain**
Start from the paper "Language Models are Few-Shot Learners" (GPT-3). Find 3 papers that cite it and are about in-context learning. For each of those, find one paper they cite that is about meta-learning. Create a Note mapping this 3-hop citation chain.
*Primary tools: semantic-scholar (x4+), extract, synthesize, create-note*

## Pre-Seeded Content Required

Before any trial, the following Notes must exist in the infospace:

1. **scaling-laws-mixed** — ~3K words mixing scaling law claims with unrelated content (for G07)
2. **rlhf-reward-modeling** — ~500 words on reward modeling in RLHF (for G12)
3. **rlhf-ppo-training** — ~500 words on PPO in RLHF (for G12)
4. **rlhf-dpo-alternatives** — ~500 words on DPO as RLHF alternative (for G12)
5. **rlhf-human-preferences** — ~500 words on human preference collection (for G12)
6. **rlhf-alignment-tax** — ~500 words on alignment tax concept (for G12)
7. **desired-tool-capabilities** — list of desired agent tool-use capabilities (for G16)

These are reset (re-seeded) before each trial.

## Notes on Randomization

- For Trial A: randomize all 18
- For Trial B: randomly split into two groups of 9. First group gets sequential review. Second group tests transfer. Multiple random splits across experimental runs would strengthen results but may not be practical given time constraints.
- For Trial C: all 18, order randomized

Minimum viable experiment: one run of each trial. Stronger: 3 runs with different random splits for Trial B.
